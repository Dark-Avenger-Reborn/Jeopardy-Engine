Field guide

How to use the redesigned control surface.

This page exists to keep the interface legible. It explains the fast path, the saved presets, and the browser-side tools that make the UI more useful without blurring the boundary between the frontend and the Jeopardy engine.

1. Load a recipe

Presets are just scan starters

Click any preset on the scan page to populate both target ranges. Your last edits are remembered locally so you can jump back without retyping.

{% for preset in presets %} {% endfor %}
2. Read the board

Use filters instead of eyeballing rows

The results page gives you search, OS filters, status filters, watchlists, and exports. That keeps the important hosts visible when the table gets busy.

  • Search by IP or OS.
  • Pin important rows with the plus icon.
  • Export the visible slice as JSON or CSV.
3. Move fast

Quick commands stay one click away

Shortcut buttons let you inject common shell commands without hunting through the terminal. They are intentionally simple so the live shell remains the source of truth.

Boundary

Separate by design

This UI is intentionally isolated from the main Jeopardy engine. It consumes the same workflow, but it does not own the core game logic.
